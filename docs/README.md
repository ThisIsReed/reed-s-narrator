# 文档导航

## 目的
本目录同时包含“需求来源、架构设计、评审意见、工作包规划、实现历史”五类材料。  
为避免重复阅读和文档职责混淆，统一按下面方式使用。

## 推荐阅读顺序
1. [`故事叙述者程序设计-核心用户观点.md`](./故事叙述者程序设计-核心用户观点.md)
   - 作用：最初的用户意图与边界约束。
   - 定位：需求来源文档，偏“为什么要做”。
2. [`初版UserStory-系统架构设计.md`](./初版UserStory-系统架构设计.md)
   - 作用：把用户意图收敛成初版验收故事。
   - 定位：需求抽象文档，偏“要满足什么”。
3. [`综合评审意见.md`](./综合评审意见.md)
   - 作用：外部评审结论、风险排序、收敛建议。
   - 定位：决策和优先级文档，偏“先解决什么问题”。
4. [`系统架构设计方案.md`](./系统架构设计方案.md)
   - 作用：目标架构、模块边界、主循环、接口与测试策略。
   - 定位：目标设计文档，偏“应该怎么实现”。
5. [`系统架构工作包拆分.md`](./系统架构工作包拆分.md)
   - 作用：按阶段拆分工作包、当前缺口、依赖顺序和后续修复计划。
   - 定位：执行规划文档，偏“接下来做什么”。
6. [`changelog.md`](./changelog.md)
   - 作用：已经落地的实现历史和验证记录。
   - 定位：事实记录文档，偏“现在已经做了什么”。

## 文档分工

| 文档 | 主要职责 | 不负责 |
|---|---|---|
| `故事叙述者程序设计-核心用户观点.md` | 记录原始目标、阶段边界、关键设计原则 | 不维护现状、不做实现规划 |
| `初版UserStory-系统架构设计.md` | 记录初版 User Story 与验收语义 | 不维护最新代码状态 |
| `综合评审意见.md` | 汇总评审共识、风险分级、修复优先级 | 不展开完整架构细节 |
| `系统架构设计方案.md` | 维护目标架构、模块边界、接口设计与推荐主循环 | 不记录逐次实现历史 |
| `系统架构工作包拆分.md` | 维护工作包、依赖、阶段判断、当前待补能力 | 不重复记录完整实现细节 |
| `changelog.md` | 维护每次实现落地、测试结果、文件变更摘要 | 不负责未来规划 |

## 使用规则
- 查“目标架构”优先看 [`系统架构设计方案.md`](./系统架构设计方案.md)。
- 查“当前缺口与下一步”优先看 [`系统架构工作包拆分.md`](./系统架构工作包拆分.md)。
- 查“哪些已经实现、怎么验证的”优先看 [`changelog.md`](./changelog.md)。
- 查“为什么这么设计”优先看 [`综合评审意见.md`](./综合评审意见.md) 和两份早期需求文档。

## 当前建议
- 把 `系统架构设计方案.md` 视为唯一的目标设计基线。
- 把 `系统架构工作包拆分.md` 视为唯一的执行规划基线。
- 把 `changelog.md` 视为唯一的实现事实基线。
