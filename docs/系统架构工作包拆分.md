## 2. 拆分原则
- 以“四层架构 + 物候独立模块 + 持久化与测试”进行纵向分包。
- 以 `docs/changelog.md` 作为当前实现状态的唯一事实来源，区分“已完成 / 部分完成 / 未开始”。
- 每个工作包同时保留目标范围、当前落地结果和后续依赖，避免文档与代码状态脱节。
